# CleanPro Telegram Bot

Окремий сервіс для працівників і адміністраторів CleanPro. Працює поруч із `backend/` та `frontend/`, використовує ту саму базу SQLite.

## Можливості

- Вхід лише через **контакт Telegram** (номер не вводиться вручну)
- Ролі з сайту: `Employee` → працівник, `Admin` → адмін
- Push про **нові замовлення** всім, хто може їх приймати
- **Прийняти замовлення** — перший забрав, у інших зникає кнопка
- Керування статусами в боті (підтвердити / виконати / повернути)
- **Статистика** працівника: сьогодні / тиждень / місяць / усього + зарплата (картка / готівка)
- **Доля** працівника (0% за замовчуванням) — без неї і без дозволу адміна приймати замовлення не можна
- **Логи** для адміна: хто, коли і що робив
- **Розсилка** одним повідомленням усім працівникам

## Налаштування

1. Створіть бота через [@BotFather](https://t.me/BotFather)
2. Вставте токен у `appsettings.Development.json`:

```json
"TelegramBot": {
  "BotToken": "123456:ABC..."
}
```

3. На сайті у користувача має бути той самий номер телефону, що в Telegram, і роль `Employee` або `Admin`.

## Запуск (коли будете готові тестувати)

```bash
cd telegram-bot/CleanPro.TelegramBot
dotnet run
```

API сайту також має працювати — саме він створює замовлення і кладе сповіщення в outbox.

## Структура

```
telegram-bot/
  CleanPro.TelegramBot/     # Worker + Telegram.Bot
    Services/               # Polling, outbox, handlers
    UI/                     # Тексти та клавіатури
    State/                  # Сесії (розсилка, зміна долі)
```

## Примітки

- Поки токен порожній — бот не стартує polling (безпечно для розробки).
- Працівників потрібно створити в БД з роллю `Employee` (через адмінку/API пізніше).
- Адмін керує долею та доступом у боті: **Працівники** → обрати людину.
